# keiba-roi-table

**「検証して」と言われたら最初にこのコマンドを実行する:**

```bash
python3 verify/bet_metrics_standard.py
```

バックテスト結果を報告する際は**必ず**このスキルのフォーマットに従うこと。

**プロジェクトパス:** `/Users/du/Documents/競馬予想システム/`

---

## 実行コマンド

```bash
# デフォルト（data/検証_新ロジック_調教あり.csv を使用）
python3 verify/bet_metrics_standard.py

# CSVを指定
python3 verify/bet_metrics_standard.py --csv data/検証_新ロジック_調教あり.csv

# 芝のみ
python3 verify/bet_metrics_standard.py --surface 芝

# ダートのみ
python3 verify/bet_metrics_standard.py --surface ダ
```

## 必須出力: 9馬券種 × 全クラス別（的中率・ROI 両方必須）

| # | 馬券種 | 点数/R |
|---|---|---|
| 1 | 単勝 | 1 |
| 2 | 複勝 | 1 |
| 3 | 馬連1-2（予想1位×2位） | 1 |
| 4 | 馬連3点BOX（予想1〜3位） | 3 |
| 5 | ワイド1-2（予想1位×2位） | 1 |
| 6 | ワイド3点BOX（予想1〜3位） | 3 |
| 7 | 三連複1-2-3（予想1〜3位） | 1 |
| 8 | 三連複1軸4頭（1位軸×2〜5位） | 6 |
| 9 | 三連複5頭BOX | 10 |

### 出力セクション（この順序・必須）

1. **総合**（2勝クラス以上・全件）
2. **クラス別**: 2勝クラス / 3勝クラス / OP / 重賞

## 禁止事項・チェックリスト

- **単勝・複勝が0%になったら即バグ** → 自作ローダーではなく必ず `load_races()` を使う
- 三連複5頭BOXのROIだけ報告して終わりにしない
- 単勝命中率だけ示して「改善」と言わない
- クラス別を省略しない
- 的中率を省略しない（ROIのみ報告は禁止）
