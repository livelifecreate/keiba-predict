## 競馬予想システム 採点ロジック

**プロジェクトパス:** `/Users/du/Documents/競馬予想システム/`
**主要ファイル:**
- `jra_scraper.py` — JRA公式サイトから出馬表を取得
- `netkeiba_race_scraper.py` — netkeiba から出馬表取得（sire/bms含む）

※指標定義: 予想TOP3(またはTOP5)の中に実際の1〜3着馬が何頭含まれるか（分母は常に3）

## 運用ルール（ブレ防止）

1. **ハイレベル重賞:** G2以上（CLASS_RANK ≥ 6）を対象
2. **コース実績の期間:** 近4戦以内のみ対象
3. **上がり3F比較:** 全出走馬の前走3Fを数値で機械的に比較（芝ダ・馬場差は無視）
4. **ペナルティ上限:** 減点合計は最大 -5.0（total properyでmax(negatives,-5.0)）
5. **クラス序列:** 未勝利(0)→1勝(1)→2勝(2)→3勝(3)→オープン/L(4)→G3(5)→G2(6)→G1(7)
6. **中央5場:** 東京・中山・京都・阪神・**中京**（中京は中央同格）
7. **ローカル5場:** 福島・新潟・小倉・函館・札幌
8. **急坂コース:** 中山・阪神・中京
9. **左回り:** 東京・中京・新潟・函館 ／ **右回り:** 中山・阪神・京都・福島・小倉・札幌
10. **海外前走の馬:** 3Fデータなし・ローカル判定対象外・特殊条件(-4)に該当
11. **昇級+距離延長同時発動:** score_all内で両方に+1点緩和を適用してからScoreBreakdownに格納
12. **前々走スコア:** prev2_high_grade_close と prev2_run_bonus は前走スコアと独立して加算

## 血統テーブル（SIRE_TABLE）

種牡馬を距離カテゴリ 1〜5 に分類:
- 1=Sprint(≤1400m), 2=Mile(≤1800m), 3=Middle(≤2200m), 4=Long(≤2600m), 5=Stayer(3000m+)
- カテゴリ差0→+1.5, 差1→+0.5, 差2→-0.5, 差3→-1.0, 差4→-1.5
- 父60%+母父(BMS)40%加重平均 → 0.5刻みに丸め

バックテストでの血統取得: `https://db.netkeiba.com/horse/ped/{horse_id}/` の `blood_table` テーブル
cells[0]=父名, cells[17]=母父名（年号・毛色を_strip_horse_name()で除去）

## 実行手順（「○○を予想して」と言われたら）

1. ユーザーが指定したレース名からrace_idを検索（find_race_ids.pyまたはdb.netkeiba日付検索）
2. 調教データURLを確認（未提供なら「追加しますか？」と聞く）
3. 採点・予想順位を出力
4. 買い目提案を出力：
   - 軸馬：人気TOP3×システム最上位の馬
   - ワイド流し4点（軸→予想TOP5）
   - 3連複TOP6フォーメーション9点（軸-予想TOP5-予想TOP6）

**重要：G1/G2の自動検出はしない。必ずレース名指定を待つ。**

```bash
# バックテスト（過去レース検証）
python3 backtest.py --race-id 202605020811 --training-url https://umasiru.com/archives/XXXXX
```
